# AGENTS.md

## What this repo is

Shell scripts that build self-contained podman/docker images with llama.cpp server + a GGUF model baked in. No runtime mounts needed. Supports NVIDIA CUDA, AMD ROCm, and Vulkan backends.

## Architecture — how the pieces connect

```
profile (profiles/*.sh)     defines REPO, FILES, DEFAULTS arrays
        |
        v
models.sh --profile NAME    downloads from HF Hub, stages GGUFs into models/
        |                    writes models/MANIFEST (REPO, COMMIT, FILES)
        |                    ALWAYS cleans models/ before staging (rm *.gguf + MANIFEST)
        v
build.sh --profile NAME     sources profile + MANIFEST
  --cuda|--rocm|--vulkan     auto-calls models.sh if models/ empty or mismatched
                             generates defaults.conf (temporary, deleted after build)
                             passes build-args to Containerfile
        |
        v
Containerfile               recreates HF cache dir structure inside image
                             COPY models/*.gguf (wildcard — copies everything in models/)
                             COPY defaults.conf → /defaults.conf
                             patches entrypoint.sh with sed (replaces __DEFAULT_MODEL__)
        |
        v
entrypoint.sh               reads /defaults.conf, merges with $@ (user flags win)
                             hardcoded infra flags: -hf, --offline, --host, --port, --metrics
```

Key: the Containerfile never references specific filenames. It copies all `*.gguf` from `models/`. If stale files are left in `models/`, they get baked into the image silently.

## Critical gotchas

- **MANIFEST must have quoted values.** `FILES="file1.gguf file2.gguf"` — unquoted multi-file values cause `source` to try executing the second filename as a command.
- **`defaults.conf` is ephemeral.** Generated by `build.sh`, copied into the image, then `rm -f`'d after build. Listed in `.gitignore` and whitelisted in `.containerignore`. Never commit it.
- **`.containerignore` is a whitelist.** Starts with `*` (ignore everything), then `!`-whitelists `models/*.gguf`, `defaults.conf`, `entrypoint.sh`. If you add a new file the image needs, you must whitelist it here or the `COPY` will fail.
- **SELinux blocks GPU access in containers.** All `podman run` commands for GPU workloads need `--security-opt label=disable`. This applies to CUDA, ROCm, and Vulkan. The CDI device flag alone is not sufficient on SELinux-enforcing hosts (Fedora/Bazzite).
- **`--network host` is required** due to a podman rootless pasta bug where IPv6 TCP connections on wildcard sockets get reset. Not optional.
- **`models.sh` has a dead `repo_to_dirname` function** (lines 40-44) that's immediately overwritten by the correct one (lines 45-48). Harmless but confusing.

## Profile format

Profiles are sourced by both `build.sh` and `models.sh`. They set three bash variables:

```sh
REPO="org/model-GGUF"           # HuggingFace repo
FILES=("model-Q4_K_M.gguf")    # array of filenames to stage
DEFAULTS=(-c 32768 -ngl 999)   # array of llama-server flags
```

`DEFAULTS` uses native llama-server flag syntax. Flag+value pairs are adjacent array elements (e.g. `-c 131072` is two elements). `build.sh` serializes these into `defaults.conf` as one flag-group per line.

## Auto-staging logic (build.sh)

When `--profile` is given, `build.sh` checks if restaging is needed by comparing:
1. `REPO` from MANIFEST vs profile
2. `FILES` from MANIFEST vs profile

If either differs, it re-runs `models.sh --profile NAME`. This handles switching models, changing quantizations, and adding/removing mmproj files. Without `--profile`, it only checks whether `models/` is empty.

## Entrypoint merge semantics

`entrypoint.sh` reads `/defaults.conf` line by line. Each line is a flag group (`--temp 0.8`). It collects flag names from `$@`, then skips any default whose flag name matches a user-supplied flag. User flags are appended last. This means user flags always win, but there's no deep parsing — it matches on the flag name string only.

## Validation

There are no tests. Validate scripts with:
```sh
bash -n build.sh && bash -n models.sh && bash -n entrypoint.sh && bash -n new-profile.sh
```

To verify a built image has the right contents:
```sh
podman run --rm --entrypoint cat IMAGE /defaults.conf
podman run --rm --entrypoint ls IMAGE /root/.cache/huggingface/hub/
```

## Device flags by backend

| Backend | Flags |
|---------|-------|
| CUDA | `--device nvidia.com/gpu=all --security-opt label=disable` |
| ROCm | `--device /dev/kfd --device /dev/dri --security-opt seccomp=unconfined --security-opt label=disable` |
| Vulkan | `--device /dev/dri --security-opt label=disable` |

These must be consistent across `build.sh` (printed run commands), `scripts/run-remote.sh` (device_flags function), and `scripts/serve.sh`.

## scripts/ directory

Utility scripts for local dev — NOT part of the container build:
- `serve.sh` — runs upstream llama.cpp image with bind-mounted HF cache (no build step)
- `run-remote.sh` — pulls and runs images from the Gitea registry, auto-detects backend from tag suffix
- `test-context.sh` — needle-in-haystack context window stress test
